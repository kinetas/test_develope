[다음 단계 권장사항]

각 기능 View 작업 시 layout.html을 기반으로 확장:

  th:replace="~{layout :: layout(
      title     = '페이지명',
      sidebar   = ~{fragments/sidebar :: gameSidebar},
      content   = ~{:: #page-content}
  )}"

권장 작업 순서:
  1. 홈/랜딩 뷰 (index.html) — layout.html 상속, 사이드바 없음
  2. 게임 플레이 뷰 (game.html) — 사이드바(인벤토리/퀘스트), 채팅 패널
  3. 캐릭터 뷰 (character.html) — stat-card 컴포넌트 활용
  4. 스토리 아카이브 뷰 (archive.html) — table 컴포넌트 활용
  5. 로그인/회원가입 뷰 — modal 또는 전용 페이지, 사이드바 없음

fragment 분리 권장:
  - templates/fragments/sidebar.html  — 사이드바 메뉴 fragment
  - templates/fragments/navbar.html   — 네비게이션 분리 (선택)
  - templates/fragments/modals.html   — 공통 모달 모음

JavaScript 다음 작업 시 연동 포인트:
  - .sidebar--mobile-open 토글 (모바일 햄버거 메뉴)
  - .modal-backdrop 동적 생성/제거
  - .loading-overlay 표시/숨김
  - data-text 속성 기반 glitch-text 자동 세팅 스크립트
