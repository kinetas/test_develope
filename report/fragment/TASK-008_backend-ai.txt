2. 설계 결정 사항
────────────────────────────────────────────────
- downloadCount: 상세 조회(GET /api/share/{id}) 시 자동 증가. 별도 download API 불필요.
- linkedSheets / linkedMaps: JSON 배열 문자열("[1,2,3]") 저장. 엔티티 Join 없이
  가볍게 참조만 저장하는 방식으로, 조회 시 프론트엔드가 파싱하여 활용.
- 공유 목록 페이지는 서버사이드 렌더링(전체+인기)과 클라이언트사이드 lazy fetch(타입별)
  혼합 방식 적용.
- 인증이 없는 사용자도 공유 목록/상세 열람 가능(public read), 등록/삭제/좋아요는 인증 필요.
