[구조 설명]
================================================================================

1. 아키텍처 레이어 (MVC + Service Layer)
   - controller/  : HTTP 요청 처리, @RestController / @Controller 위치
   - service/     : 비즈니스 로직, @Service 위치
   - repository/  : DB 접근, Spring Data JPA Repository 인터페이스 위치
   - model/       : JPA 엔티티 (@Entity) 위치
   - dto/         : 요청/응답 DTO 클래스 (API 계약 분리)
   - config/      : SecurityConfig, WebSocketConfig, JwtConfig 등 설정 클래스
   - util/        : JwtUtil, ResponseUtil 등 공통 유틸리티

2. 의존성 구성
   - spring-boot-starter-web           : MVC, REST API
   - spring-boot-starter-security      : 인증/인가 프레임워크
   - spring-boot-starter-websocket     : STOMP 기반 실시간 통신
   - spring-boot-starter-data-jpa      : ORM (Hibernate)
   - spring-boot-starter-thymeleaf     : 서버 사이드 HTML 렌더링
   - thymeleaf-extras-springsecurity6  : Thymeleaf Security Dialect
   - jjwt-api / jjwt-impl / jjwt-jackson (0.12.5) : JWT 발급·검증
   - lombok                            : Boilerplate 제거
   - h2                                : 개발용 인메모리 DB
   - spring-boot-starter-validation    : Bean Validation

3. 보안 고려사항
   - application.properties의 jwt.secret는 플레이스홀더로 처리
   - 프로덕션 배포 전 반드시 강력한 시크릿(32자 이상)으로 교체 및 환경변수화 필요
   - H2 콘솔은 web-allow-others=false (로컬만 접근 허용)
